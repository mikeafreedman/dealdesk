NEW MODULES
-----------

dd_flag_engine.py (new)
  Pure-Python DD flag generator. Seven rules, each a deal -> Optional[DdFlag]:
    R1 year_built < 1978         -> AMBER lead-paint / asbestos
    R2 no Phase I ESA on file    -> AMBER Phase I required
    R3 io_period <= 24 mo        -> AMBER short IO execution risk
    R4 dscr_yr1 < 1.0            -> RED   debt not serviceable
    R5 refi new_loan < balance   -> RED   equity injection required
    R6 lp_irr < min_lp_irr       -> AMBER LP IRR below threshold
    R7 unemployment_rate > 7%    -> AMBER tenant credit risk
  Pipeline: called in main.py after run_financials(). Failure is non-fatal
  (one rule crashing never stops the others; overall engine crash just
  logs a warning and leaves deal.dd_flags as-is). Template Section 16
  already renders deal.dd_flags; this is the first commit where those
  flags actually populate.

kpi_validator.py (new -- diagnostic scaffold for Fix 1)
  Opens the rendered xlsx with data_only=True and compares 12 key KPIs
  against deal.financial_outputs. Within $1 or 0.01% relative -> OK;
  anything wider -> warning log line with the specific Python / Excel
  pair. Call site: main.py right after populate_excel(). Produces the
  targeted cell-update list we need before a real PDF-Excel sync pass.

FINANCIALS (financials.py)
--------------------------

Fix 2 (refi equity-injection disclosure):
  When new_loan < old_balance, compute the unfloored refi math and store
  refi{i}_equity_injection_{required,amount,new_loan,existing_balance}
  on deal.provenance.field_sources so downstream (Excel warning row,
  PDF Section 13 callout, DD flag R5) can surface it. refi_proceeds
  itself stays floored at 0 so cash-flow rows remain non-negative.

Fix 6 (NOI sensitivity expense-growth dimension):
  Prior formula (1 + eg) ** 0 collapsed every row to the same opex --
  expense-growth axis was effectively dead. Now (1 + eg) ** 1 so rows
  differentiate on expense growth as intended. Fixed-base and var-base
  expenses both scale; management fee still tracks EGI.

DEBUG_EXPENSES=1 env flag (Fix 3 scaffold):
  _year_expenses() emits a full Year 1 line-by-line breakdown when the
  environment variable is set. No-op by default.

COC SENSITIVITY debug log (Fix 7 scaffold):
  One-line log on every run listing base_price, ltv_axis, and price_axis
  so the CoC sensitivity column count is visible in server_output.log.

stabilization_factor stored on pro_forma_years dicts:
  Both _build_proforma (hold strategies) and _build_proforma_for_sale
  (flip) now include stabilization_factor per year so word_builder can
  read it directly without re-deriving via _get_stabilization_factors.
